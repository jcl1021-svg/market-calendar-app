# 邮件提醒设计文档（2026-07-03）

## 目标

在日历事件**前一天的北京时间中午 12:00**，自动发一封邮件到 `jcl1021@gmail.com`，提醒即将到来的美盘事件。纯提醒用途——想看整月安排直接开网页，邮件只负责"戳一下别忘了"。

## 范围

- **提醒哪些**：日历上**全部**事件——非农（NFP）、CPI、FOMC、CME 假期（全天休市 + 提前收盘）。
- **提前量**：事件前一天，北京时间中午 12:00 发。
- **只在"明天有事"时才发**；没事件的日子不发，不打扰。
- **不做**：月初总览邮件（用户可直接看网页，冗余）；不改 `data.json`、不联网抓数据（只读现成数据）。

## 架构

复用现有 GitHub 仓库 `market-calendar-app`，新增一个**定时 Action**，与现有的静态网页、`fetch_data.py` 互不干扰。三个部件：

1. **`scripts/send_reminder.py`** — 提醒脚本（纯 Python 标准库）
   - 读同目录上一级的 `data.json`（现成数据，不联网、不改写）。
   - 算出"目标日期"= 运行当天（UTC）+ 1 天。
   - 从 `data.json.events` 里筛出 `date == 目标日期` 的事件。
   - 有事件 → 拼邮件主题+正文 → 走 Gmail SMTP 发出；无事件 → 直接退出、不发。
   - 支持 `--dry-run`（只打印邮件不发）和 `--date YYYY-MM-DD`（覆盖目标日期，便于离线测试）。

2. **`.github/workflows/reminder.yml`** — 每日定时触发
   - `cron: "0 4 * * *"`（04:00 UTC = 北京 12:00）。
   - 装 Python → 跑 `send_reminder.py`。
   - Gmail 应用专用密码从仓库 Secret 注入环境变量。

3. **保活机制** — 防 GitHub 60 天无活动自动禁用定时任务（**已定：方案 A**）
   - 用内联 git 命令做 keepalive：每次运行检查距上次提交天数，≥50 天则做一次空提交刷新计时器，任务永不睡眠。全自动，用户无需操心。（原计划用第三方动作 `gautamkrishnar/keepalive-workflow`，但它已被 GitHub 因 TOS 封禁不可用，故改为自带实现——反而无外部依赖、更干净。）
   - **已知代价**（用户已确认接受）：`git log` 里每年约多 7 条机器空提交，提交信息由该动作生成、非 `年.月.日-改动` 格式。量小，可接受。
   - （备选 B「自建时间戳文件」不更干净、C「手动重启」不可靠，均已否决。若日后介意主线噪音，可改为提交到单独分支的变体。）

## 时区与日期逻辑

- 中国无夏令时，固定 UTC+8。Action 于 04:00 UTC 运行，此刻北京为**当天** 12:00，故"明天（北京）"= UTC 当天日期 + 1 天。
- `data.json` 里的日期是**美国东部日期**（日历展示的就是它）。目标日期直接与 `event.date` 字符串比对。
- 邮件正文里给出的**北京时间**由脚本按**美国东部夏令时规则**换算：
  - EDT（3 月第 2 个周日 ~ 11 月第 1 个周日，UTC−4）：ET + 12h = 北京。
  - EST（其余，UTC−5）：ET + 13h = 北京。
  - 经济数据 8:30am ET ≈ 北京当天 20:30/21:30；FOMC 2:00pm ET ≈ 北京**次日** 02:00/03:00。

## 邮件内容

- **主题**：`【美盘日历】明天 MM/DD（周X）：<事件名，多个用、连接>`
  - 假期附标记：全休 → `（休市）`；提前收盘 → `（提前收盘）`。
- **正文**：每个事件一条项目符号
  - 经济数据：英文名 / `美东 <time> ET（北京 <换算> 约 HH:MM）` / 中文说明。
  - 全休假期：`<英文名> — 全天休市` / 中文说明（无时间行）。
  - 提前收盘：`<英文名> — 提前收盘` / 中文说明。
  - 底部附日历网址 `https://jcl1021-svg.github.io/market-calendar-app/`。
- 英文名、中文说明**直接复用 `data.json` 现成字段**（`title` / `desc`），不另编。
- 同一天多个事件 → 主题并列、正文逐条列出。

**样例（经济事件）**
```
主题：【美盘日历】明天 07/29（周三）：FOMC

明天有以下美盘事件，注意波动 / 交易安排：

• FOMC Statement
  美东 2:00pm ET（北京 07/30 约 02:00）
  联邦公开市场委员会 (FOMC) 利率决议与政策声明.

日历：https://jcl1021-svg.github.io/market-calendar-app/
```

**样例（假期）**
```
主题：【美盘日历】明天 12/25（周五）：Christmas Day（休市）

• Christmas Day — 全天休市
  美股与股指期货全天休市.

日历：https://jcl1021-svg.github.io/market-calendar-app/
```

## 发信方式

- Python 标准库 `smtplib` + `email.message`，Gmail SMTP（`smtp.gmail.com:465` SSL）。
- 登录用 `jcl1021@gmail.com` + **应用专用密码**（Google 账号后台生成，需已开两步验证）。发件人 = 收件人 = 该邮箱。
- 密码存 GitHub 仓库 Secret `GMAIL_APP_PASSWORD`，Action 运行时注入，**绝不写进代码/仓库**。

## 错误处理

- `data.json` 读取失败 / 格式错 → 脚本报错退出（Action 标红，可在 GitHub 看到失败通知）。
- SMTP 发送失败 → 抛异常、退出码非 0，Action 标红。
- 无匹配事件 → 正常退出（退出码 0），不发信、不算失败。

## 可靠性说明（已与用户确认的取舍）

- **风险 1（60 天自动睡眠）**：保活机制彻底解决。
- **风险 2（GitHub 偶发延迟/跳过单次触发）**：延迟对"提前一天中午"无影响；整次跳过极少见，用户接受该残余风险（不加月初总览兜底，因可随时看网页）。
- 不追求"绝对不漏"——已确认没有免费自动系统能保证 100%，此方案在够稳与够简单之间取平衡。

## 测试

- `send_reminder.py` 的**取日期、筛事件、拼正文、北京时间换算**均为纯逻辑，可离线测试：
  - `--dry-run --date <某有事件日的前一天>` 打印邮件，人工核对主题/正文/时区。
  - 覆盖：单事件、多事件同日、全休假期、提前收盘、无事件（不发）、EDT 与 EST 两种换算。
- 发信环节用 `--dry-run` 隔离，不依赖真实 SMTP 即可验证全部格式逻辑。

## 用户需自行完成的一次性配置

1. Google 账号开两步验证 → 生成**应用专用密码**。
2. 在 GitHub 仓库 `Settings → Secrets and variables → Actions` 新建 Secret `GMAIL_APP_PASSWORD`。
（这两步涉及登录用户账号，需用户在网页后台操作，Claude 会分步指引。）
